### 1.5 两阶段执行策略（重要）

由于完整训练耗时较长（每个 epoch 约 50 分钟，50 epoch 需数十小时），本模块采用**两阶段策略**：

**阶段一：快速验证** — 同时使用了**数据采样 + 少量 epoch** 双重加速
- 数据采样：仅取 5000 条训练交互、1000 条验证、2000 条测试
- 少量 epoch：每组训练仅 10 个 epoch
- 目的：验证代码能否跑通、loss 是否下降、超参/α 的趋势是否正确

| 脚本 | 措施 | 预计耗时 |
|------|------|---------|
| `train_ncf_first.py` | 采样 5000 条 + 10 epoch | **~2-5 分钟** |
| `train_ncf_review_first.py` | 采样 5000 条 + 10 epoch | **~2-5 分钟** |
| `train_ncf_hyper.py` | 采样 5000 条 + 10 epoch/组 × 8组 | **~15-40 分钟** |
| `train_ncf_review_alpha.py` | 采样 5000 条 + 10 epoch/α × 4 | **~8-20 分钟** |

**阶段二：正式训练** — 使用**全量数据 + 完整 50 epoch**

| 脚本 | 措施 | 预计耗时 |
|------|------|---------|
| `train_ncf_best.py` | 全量数据 + 50 epoch | 数十小时 |
| `experiments/exp1_baseline.py` | 全量数据 + 50 epoch | 数十小时 |
| `experiments/exp2_semantic.py` | 全量数据 + 50 epoch | 数十小时 |
| `experiments/exp3_graph.py` | 全量数据 + 50 epoch | 数十小时 |

**推荐流程**：
1. 先跑完**阶段一** 4 个脚本（总耗时约 **30 分钟到 1 小时**），确认代码无误、loss 正常下降
2. 确认没问题后，再跑**阶段二**的正式训练，可以挂机过夜

> **注意**：如果使用 GPU，阶段一每个脚本可能仅需几十秒。以上时间为 CPU 环境下的粗略估计。

## 2. NCF 模型训练与调参

### 2.1 模型架构说明

NCF（Neural Collaborative Filtering）使用双通路架构：

```
                     ┌──────────────────────┐
                     │     Score Output      │
                     │   (Sigmoid, 0~1)      │
                     └──────────┬───────────┘
                                │
                     ┌──────────┴───────────┐
                     │    Concat + Linear    │
                     └──────────┬───────────┘
                                │
            ┌───────────────────┴───────────────────┐
            │                                       │
   ┌────────┴────────┐                    ┌─────────┴─────────┐
   │   GMF Path      │                    │     MLP Path      │
   │ (u_gmf * i_gmf) │                    │ [64→32→16 ReLU]   │
   └────────┬────────┘                    └─────────┬─────────┘
            │                                       │
   ┌────────┴────────┐                    ┌─────────┴─────────┐
   │ user_emb_gmf    │                    │  user_emb_mlp     │
   │ item_emb_gmf    │                    │  item_emb_mlp     │
   └────────┬────────┘                    └─────────┬─────────┘
            │                                       │
            └───────────────────┬───────────────────┘
                                │
                     ┌──────────┴──────────┐
                     │   user_id, item_id   │
                     └─────────────────────┘
```

**代码位置**：[src/ncf.py](file:///c:/homework/dm-final-explainable-rec/src/ncf.py)

**关键类**：

| 类/函数 | 说明 |
|---------|------|
| `NCF(n_users, n_items, embedding_dim, mlp_layers)` | 模型主体，继承 `nn.Module` |
| `train_ncf(model, train_df, val_df, config, n_items, device)` | 训练循环，含每 epoch 重采负样本 + 早停 |
| `NCF.recommend(user_id, n_items, k, exclude, device)` | 生成 Top-K 推荐列表 |

## 3. NCF+Review 融合模型训练

### 3.1 模型架构说明

NCF+Review 在 NCF 基础上引入评论文本的语义信息：

```
                    ┌───────────────────────┐
                    │      Score Output      │
                    │    (Sigmoid, 0~1)      │
                    └───────────┬───────────┘
                                │
                    ┌───────────┴───────────┐
                    │     Fusion MLP         │
                    │  [64→32→1] Sigmoid     │
                    └───────────┬───────────┘
                                │
                    ┌───────────┴───────────┐
                    │    Weighted Fusion     │
                    │ (1-α)*behav + α*sem   │
                    └───────────┬───────────┘
                                │
            ┌───────────────────┴───────────────────┐
            │                                       │
   ┌────────┴────────┐                    ┌─────────┴─────────┐
   │  Behavior Path  │                    │   Semantic Path   │
   │  (NCF MLP)      │                    │  (SBERT Proj)     │
   │  [64→32→16]     │                    │  384→64 Linear    │
   └────────┬────────┘                    └─────────┬─────────┘
            │                                       │
   ┌────────┴────────┐                    ┌─────────┴─────────┐
   │ user_emb (ID)   │                    │  user_emb (SBERT) │
   │ item_emb (ID)   │                    │  item_emb (SBERT) │
   └────────┬────────┘                    └─────────┬─────────┘
            │                                       │
            └───────────────────┬───────────────────┘
                                │
                     ┌──────────┴──────────┐
                     │   user_id, item_id   │
                     └─────────────────────┘
```

**融合公式**：

```
final_vec = (1 - α) * behavior_vec + α * semantic_vec

其中 behavior_vec = MLP([user_id_emb, item_id_emb])
      semantic_vec = Linear(user_sbert_emb) + Linear(item_sbert_emb)
      α 控制语义信息的权重（默认0.3）
```

**代码位置**：[src/ncf_review.py](file:///c:/homework/dm-final-explainable-rec/src/ncf_review.py)

## 4. 实验1：基线对比

### 4.1 实验设计

**研究问题**：NCF 相比传统方法是否有显著提升？

**对比方案**：

| 模型 | 核心机制 | 对照目的 |
|------|---------|---------|
| Popularity | 按交互次数排序 | 最低基线 |
| UserCF | 用户间协同过滤 | 传统记忆型方法 |
| ItemCF | 物品间协同过滤 | 传统记忆型方法 |
| **NCF** | 神经网络拟合交互 | **模块B核心模型** |

**为什么实验1放在这里？**

实验1的脚本 [experiments/exp1_baseline.py](file:///c:/homework/dm-final-explainable-rec/experiments/exp1_baseline.py) 属于**整个项目的基础实验**，但和模块B有直接关系：
- 它训练了 **NCF** 模型（模块B的核心模型之一），调用 [`src/ncf.py`](file:///c:/homework/dm-final-explainable-rec/src/ncf.py) 的 `NCF` 和 `train_ncf`
- 它的结果作为**后续实验的基线参照**——实验2（语义贡献）和实验3（图模型对比）都需要对比 NCF 的指标，才能看出改进效果
- 它使用的数据与模块B完全一致（`data/processed/train.csv`、`val.csv`、`test.csv`）

## 9. 常见问题与解决方案

### 9.1 数据文件找不到

**现象**：
```
FileNotFoundError: [Errno 2] No such file or directory: 'data/processed/train.csv'
```

**原因**：模块A的数据处理 Pipeline 未运行。

**解决**：
```powershell
# 运行数据 Pipeline（需要原始数据 data/raw/Electronics_5.json.gz）
python src/preprocess.py
```

### 9.2 SBERT Embedding 缺失

**现象**：
```
FileNotFoundError: data/processed/user_emb.npy
```

**原因**：模块D的 SBERT 编码未执行。

**解决**：联系模块D运行 Pipeline Stage 8，或手动执行：
```powershell
python -c "
from src.preprocess import encode_sbert
encode_sbert()
"
```

### 9.3 CUDA 显存不足

**现象**：
```
RuntimeError: CUDA out of memory.
```

**解决**：
- 减小 `batch_size`（config.py: `model.batch_size: 256`）
- 减小 `embedding_dim`（设为 32）
- 使用 CPU 训练（`device='cpu'`）

### 9.4 NCF 训练不收敛

**现象**：loss 在 10+ epoch 后仍不下降

**检查清单**：
1. embedding 初始化正常？（`nn.init.normal_(std=0.01)`）
2. learning rate 合适？（尝试 0.001 → 0.0005）
3. 负采样 ratio 足够？（neg_ratio >= 4）
4. 数据稀疏度过高？（stats.json 中 sparsity > 0.995 时需检查）

### 9.5 LightGCN 训练慢

**现象**：每个 epoch 非常慢

**原因**：稀疏矩阵乘法在大图上的开销。

**解决**：
- 减小 `lightgcn_layers`（3 → 2）
- 增大 `batch_size`（512 → 1024）
- 使用 GPU 加速

### 9.6 NCF+Review 评估报错

**现象**：
```
TypeError: recommend() missing required arguments
```

**原因**：`NCFReview.recommend` 需要 SBERT embedding，但 `evaluate_model` 不会自动传入。

**解决**：确保训练后调用 `model.set_review_embeddings(user_emb, item_emb)`。本模块已修复此问题（见 [src/ncf_review.py](file:///c:/homework/dm-final-explainable-rec/src/ncf_review.py) 中的 `set_review_embeddings` 方法和 `recommend` 的后备逻辑）。
